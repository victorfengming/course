文档排版和图书出版方案比较（了解）

  • 网页排版使用HTML
  • 电子文档排版，传统文档多用Word，计算机类文档多用Markdown

Markdown介绍

  • Markdown是一种文本标记语言，用于快速文档排版

为什么使用Markdown？

  • 应用广泛：技术类／写作类应用的流行方案

    • http://segmentfault.com
    • http://cnblogs.com
    • http://html-js.com
    • http://oschina.net
    • http://jianshu.com

  • 轻量／简洁

    • Markdown文件为纯文本文件，后缀名为 .md
    • 介于Word和HTML之间
      • 比起Word，Markdown是纯文本，轻量
      • 比起HTML，Markdown排版速度快，易读、易写、易修改

Markdown优点

  • 简单、流行、B格高且使用广泛，排版功能足够使用（不足之处可用第三方插件、HTML、CSS弥补）

Markdown缺点

  • 太简单，很多复杂的专业排版样式功能无法实现，只适用于要求不高的电子文档排版场合（电子书、办公、技术文档等）

Markdown编辑器

  • 任意文本编辑器（能编辑，不能预览，只能脑补排版效果）
  • 专业Markdown编辑器：支持Markdown文档排版后预览，适合文字工作者
    • MAC：mou，macdown
    • Windows：MarkdownPad
  • 带Markdown预览功能的专业代码编辑器：适合编程、分析等理工类应用
    • VScode、Atom等原生支持预览（安装插件后功能更强）
    • Sublime Text、Pycharm等安装插件后支持预览
    • Jupyter Notebook：原生增强支持，可直接运行代码并输出结果

注意：

  • 不同编辑器预览和生成其他格式文档时，对Markdown语法的解析和支持可能不一致
  • VScode编辑器在MACOS系统下隐藏退格空字符BUG：
    • 当切换为中文输入法如搜狗拼音，在输入框未消失时退格删除输入框字符，删到最后一个字符时会生成一个退格空字符，在VScode内占位但不显示，存在却看不到，导致代码执行和文档显示错误，例如 “url举例” 中有一个退格符。
    • 解决方式：VScode - 左下角 设置 - 用户设置 - 加入下面配置
{
    // 控制编辑器是否应呈现控制字符,默认false改为true
    // 因为MAC下用中文输入法输入框未消失就退格删除时会在最后一个字符出现隐藏的退格符，看不见却存在导致显示和执行代码问题
    // 设置后此类隐藏符号在VScode中显示为小号 BS
    "editor.renderControlCharacters": true,
}

Markdown预览和输出HTML文档

基本方法：

  • 这里以VScode编辑器为例
    • 方法1：VScode编辑器原生支持Markdown预览，但不支持输出HTML，功能薄弱
      • 使用原生md预览：新建或打开md文档，编辑器右上角-打开侧边预览
    • 方法2：推荐，编辑器安装Markdown Preview Enhanced插件
      • 预览md文档：快捷键CTRL+Shift+P(Mac:Cmd+Shift+P)（或者编辑器左下角设置按钮-命令面板），输入Markdown Preview Enhanced:Open Preview，回车打开预览
      • 输出HTML：预览页面鼠标右键-HTML-HTML(offline)，HTML文档会自动保存到md文件同级目录（注意右键其他选项不安装对应库不能使用）
      • 输出PDF：最简单的方式，Chrom浏览器打开HTML文档，右上角栏目-打印-打印为PDF

高级方法：

  • 使用基于Haskell的Pandoc转换任意电子文档格式
  • 推荐：使用基于Node.js的Gitbook库自动生成HTML/pdf/mobi格式的电子书文档
  • 使用基于Python的Sphinx库生成电子书文档

1：在Markdown中直接写HTML/CSS/JavaScript

不在md范围内的标记或样式，或者用Markdown很难实现的标记或样式，可以直接写HTML/CSS/JavaScript语言实现

你完全可以把md文档当成HTML文档在里面写原生HTML/CSS/JS代码（注意环境支持度）

  • HTML： 所有编辑器都支持
  • CSS： 大部分支持（Jupyter Notebook只支持行内样式，可通过魔术命令完全支持）
  • JS： 基本都不支持，如想写JS可导出HTML文档后自行修改（Jupyter Notebook可通过魔术命令支持）

  background:#f00;
}
</style>

</script>

2：在Markdown中载入基于文本的公式

上面图片标签讲了怎样载入基于图片的公式

怎样载入基于文本的公式？
  • Jupyter Notebook：原生支持，执行步骤1和3
  • VSCode：Markdown Preview Enhanced插件预览，原生支持，执行步骤1和3
  • 外部使用，例如自己新建一个HTML文档，写入数学公式，执行步骤1/2/3

1：获取公式代码

  • myscript网站可以手写公式，自动转为LaTeX语法：
  • http://webdemo.myscript.com/views/math.html#

2. HTML文档载入MathJax.js

  • MathJax是一个渲染公式的JS库，LaTex语法书写
  • 使用MathJax显示公式，文档预览必须载入本JS库
  • 点此下载MathJax.js

注意：

  • 原生支持的环境默认已载入库，不要自己载
  • 载入JS库路径问号后面的参数必须带
CDN远程载入
<script src="http://cdn.mathjax.org/mathjax/latest/MathJax.js?config=TeX-AMS-MML_HTMLorMML"></script>

或下载JS库后本地载入
<script src="images/MathJax.js?config=TeX-AMS-MML_HTMLorMML"></script>

3. 将LaTeX公式写入$ $(不换行)或$$ $$(换行)之间，空格分隔

$$ P(A \mid B) = \frac{P(B \mid A) \, P(A)}{P(B)} $$

效果(需启用JS支持才可预览)：

$$ P(A \mid B) = \frac{P(B \mid A) \, P(A)}{P(B)} $$
