Pandas数据存取

数据的输入和输出是Pandas的基础操作

Pandas可以存取多种介质类型数据：常见的有：

  • 文本类数据
    • csv
    • JSON
  • 二进制磁盘数据
    • Excel
    • pkl
    • HDF5
  • 数据库
    • SQL（略）
  • Web API数据
    • HTML
  • 其他
    • 内存
文本类数据文件读入Pandas时会自动推断每列数据类型（类型推断）并转化。
二进制类数据文件会在格式中存储数据类型

对Pandas不能直接支持或不方便使用的数据格式，
可以使用支持软件将其转为csv或xlsx格式后使用Pandas读写，如SPSS文件

Pandas存取CSV

CSV

  • Comma-Separated Values，逗号分隔值
  • 以纯文本形式存储表格数据的一种格式
  • 二维表格数据结构

CSV是一种简单、通用的文件格式，常用于在不同程序和环境之间中转表格数据， 这些程序本身是在不兼容的格式上进行操作的（往往是私有的和/或无规范的格式）， 基本所有数据类软件和环境都支持读写CSV文件

写入CSV

df.to_csv('foo.csv')
  • 数据有中文，编码可以修改，默认ascii，改为utf-8
  • 写入不修改编码则读取时需要修改
a.to_csv('foo.csv')

df['Chinese'] = '中,文'

df.to_csv(
    'foo2.csv', # 存入数据
    index=False, # 不存储行索引
    header=False, # 不存储列索引
    columns=['B', 'C', 'A'],  # 保存指定列
    encoding='utf-8' # 修改编码
)

读取CSV

pd.read_csv('foo.csv')  # 注意中文内容编码问题导致的错误

更多参数

pd.read_csv(
pd.read_csv(
    'foo.csv', # 文件名
    sep=',', # 指定分隔符，csv默认逗号，如果是table表格数据一般为 \t

    header=0, # 列索引，默认0将第一行设为表头（其他行号也可），None不将第一行设为表头（列索引），[0,1]列表可将多行同时设为表头（层次化索引）
#     index_col=['name','age'],  # 行索引，默认None无行索引，列索引0,1或自定义'name'等将某列设为行索引，[0,1],['name','age']层次化索引使用列表
#     names=['a','b','c','d'], # 不使用csv表头，自定义表头,结合header=None

#     usecols=[0,1,2,3], # 读取指定列,或使用自定义索引
#     nrows=3, # 读取前几行
#     skiprows=2,  # 从表格开始算起，忽略的行，[2,3,4]跳过文件第2/3/4行
#     skipfooter=2,  # 从表格末尾算起忽略的行，必须配合engine='python'否则会报警告
#     engine='c',  # 引擎。c更快，python更完善

    na_values=['male'],  # 将csv中某些字符替换为空值
    keep_default_na=True,  # 默认True同时使用系统自带空值替换和自定义空值，如 NA,N/A等，False只使用自定义空值

    encoding='gbk', # 编码，需要根据文本编码修改，默认utf-8,可以指定为GBK/ascii
)
)
  • csv文件内有汉字等特殊符号时，csv文件编码应为utf-8(无BOM)可默认正常读取，如果编码是ANSI，加参数encoding='gbk'
  • 数据内有逗号时，左右加英文半角双引号，可以正常解析

合并时间列及自定义某列为行索引

parse_dates

尝试将数据解析为日期
可以使用列表指定需要解析的一组列名，如果列表元素为字典包含的列表或元组，会将多个列组合到一起再解析日期解析（如日期和时间分别在两个列的情况）

keep_date_col

如果连接多列解析日期，保存参与连接的列，默认False

'''
data,time,time2,name,age
20100101,000000,00:00:00,"张三",18
20100101,230000,23:00:00,"李,四",28
'''

z = pd.read_csv(
    'aaa.csv',
#     parse_dates=[['data','time2']],  # 将两列合并解析为时间格式，默认列名合并，_分隔
    parse_dates={'s': ['data','time2']},  #将两列合并解析为时间格式，自定义列名
    keep_date_col=True,  # 保留合并前的列
    index_col='s', #将某列设为行索引
)
z
z.info()

Pandas存取JSON

JSON（JavaScript Object Notation）是通过HTTP请求在Web浏览器和其他应用程序之间发送数据的标准格式之一

JSON和CSV比较：

  • JSON是多维数据文件，CSV是二维数据文件
  • JSON数据比较冗余，体积较大，CSV精简，体积小
  • JSON多用于WEB数据交互，CSV多用于计算机软件程序数据交互

JSON数据例子：

obj = """
{"name": "Wes",
 "places_lived": ["United States", "Spain", "Germany"],
 "pet": null,
 "siblings": [{"name": "Scott", "age": 30, "pets": ["Zeus", "Zuko"]},
              {"name": "Katie", "age": 38,
               "pets": ["Sixes", "Stache", "Cisco"]}]
}
"""

语法上，除其空值null和其他的细微差别（如列表末尾不允许存在多余的逗号，对象中所有的键都必须是字符串）外，JSON非常接近Python原生数据结构。

都是由：多维的字典、数组（列表）、字符串、数值、布尔值以及null 组成

JSON转为DataFrame

  • JSON数据可以由原生Python直接读入并转为字典数据，可以将字典数据直接转化为DataFrame
  • 或者使用Pandas自带的存取JSON方法
# 存入JSON
a.to_json('foo.json')

# 读取JSON
pd.read_json('foo.json')

Pandas存取pkl

pickle是Python的标准模块，实现数据的序列化和反序列化

通过pickle可以将Python的对象保存至二进制pkl文件中（例如将训练好的机器学习模型保存至pkl中方便下次读取）

实现数据的高效二进制格式存储最简单的办法之一是使用Python内置的pickle序列化。pandas对象都有一个用于将数据以pickle格式保存到磁盘上的to_pickle方法

# 存入pickle
a.to_pickle('a.pkl')

# 读取pickle
pd.read_pickle('a.pkl')

注意：pickle仅建议用于短期存储格式。原因是很难保证该格式永远是稳定的；今天pickle的对象可能无法被后续版本的库读取出来

使用HDF5格式

pandas内置支持两个二进制数据格式：HDF5和MessagePack。

HDF5是一种存储大规模科学数组数据的非常好的文件格式。 可以被作为C标准库，带有许多语言的接口，如Java、Python和MATLAB等

HDF5中的HDF指的是层次型数据格式（hierarchical data format）

每个HDF5文件都含有一个文件系统式的节点结构，它使你能够存储多个数据集并支持元数据。
与其他简单格式相比，HDF5支持多种压缩器的即时压缩，还能更高效地存储重复模式数据。

虽然可以用PyTables或h5py库直接访问HDF5文件，pandas提供了更为高级的接口，
可以简化存储Series和DataFrame对象。HDFStore类可以像字典一样，处理低级的细节

frame = pd.DataFrame({'a': np.random.randn(100)})
frame

# 简单应用

# 存入
frame.to_hdf('mydata1.h5', 'obj3')  # 必须带key
# HDFStore支持两种存储模式，'fixed'和'table'。后者通常会更慢，但是支持使用特殊语法进行查询操作：
frame.to_hdf('mydata2.h5', 'obj3', format='table')

# 读取
pd.read_hdf('mydata2.h5')  # 只能在HDF5文件只存储1个key数据时使用，否则会出错
pd.read_hdf('mydata2.h5', 'obj3', where=['index < 5'])  # 存储多个数据时，读取需要指定key，table格式存储的可以加判断条件

# 简单应用读取时如果文件内有多个未知key，会出错
##################################
# 复杂应用

# 创建或读取HDF5文件（没有就创建，有就读取）
store = pd.HDFStore('mydata3.h5')
store

# 添加数据,Dataframe
store['obj1'] = frame
store

# 添加Series
store['obj1_col'] = frame['a']
store

# HDF5文件中的对象可以通过与字典一样的API进行获取：
store['obj1']

# put是store['obj2'] = frame方法的显式版本，允许我们设置其它的选项，比如格式
store.put('obj2', frame, format='table')
store
store.select('obj2', where=['index >= 10 and index <= 15'])  # table模式保存的猜可以加判断条件

注意：

  • 如果文件被占用重启Python内核
  • HDF5不是数据库。它最适合用作“一次写多次读”的数据集。虽然数据可以在任何时候被添加到文件中，但如果同时发生多个写操作，文件就可能会被破坏。
